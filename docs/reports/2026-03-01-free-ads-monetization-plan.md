# 2026-03-01 Free+Ads Monetization Plan (Kids Learning App)

## 방향 전환
- monetization: **유료 구독 -> 무료 + 광고 수익**
- 목표: 결제 장벽 제거로 유입/활성 확보 후, 정책 준수형 광고로 수익화.

## 핵심 판단
- 가능성: **있음 (조건부)**
- 전제: 아동 앱은 광고 정책 제약이 크고, 개인화 광고 의존이 어렵기 때문에
  - 고품질 유지율
  - 반복 방문
  - 안전한 광고 위치 설계
  를 동시에 충족해야 함.

## 정책/리스크 (필수 준수)
1. FTC COPPA
- 아동 데이터 최소 수집, 보호자 관점 고지/통제 필요.
- https://www.ftc.gov/business-guidance/privacy-security/childrens-privacy

2. Google Play Families
- 아동 대상 앱은 허용된 광고/SDK/콘텐츠 정책 준수 필요.
- https://support.google.com/googleplay/android-developer/answer/9893335

3. Apple Kids Category
- Kids 카테고리 앱의 추적/광고/외부 링크/개인정보 처리 제약 존재.
- https://developer.apple.com/app-store/review/guidelines/#kids-category

## 광고 수익 검증 공식 (객관)
- **Revenue/day = DAU x (Impressions per DAU) x eCPM / 1000**
- 의미:
  - DAU가 작으면 수익이 거의 안 남음
  - 노출을 무리하게 늘리면 리텐션이 깨짐
  - 따라서 `리텐션 유지 + 적정 노출`의 균형이 핵심

## 제품 내 광고 배치 원칙 (아동 친화)
1. 학습 진행 중 전면 광고 금지
- 문제풀이/트레이싱/매칭 도중 방해 금지

2. 허용 위치
- 홈 화면 하단 배너
- 리워드/결과 화면 배너
- 부모 화면 배너(보호자 전용)

3. 빈도 제한
- interstitial/rewarded 사용 시 최소 간격(예: 3분+) 강제
- 어린이 경험 저하 시 즉시 낮춤

## 30일 KPI (무료+광고 모델)
1. 활성/리텐션
- 7일 활성일수 >= 4일
- 2주 유지율 프록시 >= 70%

2. 학습 품질
- 일평균 학습시간 >= 15분
- 루틴 완료 >= 2개/일
- 미션 달성률 >= 70%

3. 광고 품질
- 광고 노출 증가 후 D7 유지율 하락 <= 5%p
- 부모 불만(광고 관련) 비율 <= 3%

## 단계별 실행
1. Week 1: 안전 배치 적용
- 홈/리워드/부모 화면만 광고 허용

2. Week 2: 노출 실험
- 배너 위치/크기 A/B (학습경험 영향 측정)

3. Week 3: 리텐션 보호
- 광고 증분 대비 D1/D7 하락 구간 자동 차단

4. Week 4: 확장/축소 결정
- KPI 충족 시 트래픽 확장
- 미충족 시 광고 강도보다 콘텐츠 루프/온보딩 먼저 개선

## 이번 코드 반영
- `js/app.js`
  - 상용화 검증 가이드를 유료 전환 기준에서 **광고 수익 공식 기준**으로 변경
  - 배너 노출 대상에 `parent` 화면 추가
  - 검증 카드 액션 문구를 무료+광고 모델로 정렬

## 결론
- "무료 + 광고"는 가능하지만, 아동앱 특성상 **정책 준수 + 리텐션 보호**가 수익보다 먼저다.
- 즉시 목표는 ARPU 극대화가 아니라, **안전한 DAU/리텐션 기반**을 만드는 것이다.
